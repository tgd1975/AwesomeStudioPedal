## Description

All markdown files emitted by the task-system generators — primarily
`scripts/housekeep.py` and any helpers it calls — must be markdownlint-correct
**by construction**. Today the generated index files (`docs/developers/tasks/OVERVIEW.md`,
`EPICS.md`, `KANBAN.md`, and `docs/developers/ideas/OVERVIEW.md`, plus any other
autogenerated `.md` produced by the task system) can land with lint issues that
only get cleaned up if a human runs `/fix-markdown` afterwards.

The fix belongs in the generators, not in a downstream formatter. Audit every
code path that writes markdown, identify the rules they violate (heading levels,
blank lines around lists / headings / fenced blocks, consistent list markers,
trailing whitespace, single trailing newline, table spacing, etc.), and adapt
the emitters so their output passes `/lint` immediately.

## Audit

| Generator | Function | Risk surface | Fix |
|-----------|----------|--------------|-----|
| `housekeep.py` | `_flat_list_section` | task `title:` rendered raw into per-epic flat-list table | `_md_cell` on title + effort |
| `housekeep.py` | `generate_epics_md` (index row) | epic `title:` rendered raw into index table | `_md_cell` on epic_title (anchor still uses raw text) |
| `housekeep.py` | `generate_epics_md` (per-epic heading) | epic `title:` rendered into `## EPIC-NNN: …` heading | `_md_cell` on epic_title |
| `housekeep.py` | `generate_kanban_md` (`card`) | task `title:` rendered into Mermaid `["…"]` label | strip `<` `>` (mermaid label hazard, not just MD033) |
| `housekeep.py` | `generate_kanban_md` (index/heading) | epic name rendered into anchor link text and `## …` heading | `_md_cell` on epic |
| `update_task_overview.py` | `task_table_rows` | active/paused/open table row | `_md_cell` on title + effort + complexity |
| `update_task_overview.py` | closed-tasks table loop | closed table row | `_md_cell` on title + effort |
| `update_task_overview.py` | `generate_release_overview` | per-release archive snapshot row | `_md_cell` on title + effort + complexity |
| `update_idea_overview.py` | `render_overview` (open table) | open ideas row | `_md_cell` on title + description (replaces the old `\|`-only escape) |
| `update_idea_overview.py` | `render_overview` (archived table) | archived ideas row | `_md_cell` on title |

**The single rule the helpers learn:** `_md_cell(text)` escapes `<` → `&lt;`,
`>` → `&gt;`, `|` → `\|`. This guards against MD033 (no-inline-html) on
titles like `archive/<version>/` and against pipe-bearing titles that would
split table cells. The helper is duplicated across the three modules to
avoid an import cycle (housekeep is the orchestrator, the two `update_*`
scripts are the leaves).

**Mermaid labels** (`generate_kanban_md.card`) strip `<`/`>` outright rather
than entity-escape — entities don't render as glyphs inside Mermaid labels,
so the safer fix is to drop the brackets from the label (`/ts-task-active`
nags-style titles already have `"`/`` ` `` stripped for the same reason).

**Why most rules don't apply to these generators.** The bodies are built as
`list[str]` and joined with `\n`, with explicit blank-line entries flanking
every heading, fence, and table — so MD022, MD031, MD032, MD036, MD040 are
already satisfied by construction. Trailing newline (MD047) is enforced by
`"...".write_text("\n".join(lines) + "\n")`. The remaining family of rules
that *could* fire — MD033 and pipe-injection — was the actual gap.

## Test Plan

**Host tests** (`make test-host`):

This task touches Python scripts, not C++ — so the C++ host-test layer does not
apply. Verification is via Python unittests + the lint pipeline:

- `scripts/tests/test_housekeep.py::TestMdCellEscape` — unit tests for the
  shared escape helper, including the double-application-is-safe guarantee.
- `scripts/tests/test_housekeep.py::TestGeneratedMarkdownIsLintClean` —
  builds a fixture tree with hostile titles (`<version>`, `|`-bearing),
  runs the generator, and asserts `markdownlint-cli2` reports zero findings
  on `EPICS.md` and `KANBAN.md`. Includes an idempotence test that re-runs
  the generators and asserts byte-stable output.
- `scripts/tests/test_update_idea_overview.py::TestGeneratedOverviewIsLintClean` —
  same shape, applied to the ideas `OVERVIEW.md` for hostile title and
  description.
- Manually: on the live tree, `/housekeep` followed by `markdownlint-cli2`
  on the four index files + four archive snapshots reports zero findings.

## Notes

- Likely culprits to investigate: heading-level jumps, missing blank lines
  around lists/tables/fences, inconsistent list-marker style (`-` vs `*`),
  trailing spaces, missing final newline, bare URLs, and table column
  alignment.
- The same audit should cover any future autogenerated markdown — if the task
  system grows a new generator, it inherits the same contract.
- Coordinate with `/fix-markdown` and `/lint` so they agree on the rule set
  the generators target. If the linter config needs tightening, do that here
  too — the goal is one canonical definition of "correct" that generators
  satisfy.
- Out of scope: hand-written markdown under `docs/`. Those are covered by the
  existing `/lint` workflow and are not the subject of this task.
