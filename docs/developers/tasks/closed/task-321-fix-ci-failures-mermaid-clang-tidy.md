## Description

Two GitHub Actions workflows have been failing on `feature/config-extensions`
(most recently on commit `a5dbd19`, TASK-303):

- **CI / Mermaid Diagram Lint** — exit 1 in *Run Mermaid validation script*
  ([run 25131913472](https://github.com/tgd1975/AwesomeStudioPedal/actions/runs/25131913472)).
- **Static Analysis / clang-tidy** — exit 123 in *Run clang-tidy*
  ([run 25131913531](https://github.com/tgd1975/AwesomeStudioPedal/actions/runs/25131913531)).

Both failures slipped past local checks before push, so the deeper question is
why our pre-commit hook (or equivalent local workflow) did not catch them. We
have local skills `/lint`, `/clang-tidy`, and `/format`, plus a pre-commit hook
in `scripts/install_git_hooks.sh` — at least one of these is either not
running, not wired into the hook, or being silently bypassed.

The task is to (1) identify and fix the offending diagram(s) and source
file(s), and (2) close the local-vs-CI gap so future commits cannot land in
this state without an explicit, named bypass.

## Closing Comment (2026-04-30)

**How each failure escaped local checks:**

- **Mermaid Diagram Lint.** Root cause was in
  `awesome-task-system/scripts/housekeep.py` — kanban card labels stripped only
  `"`, not backticks. Task TASK-318's title `` `/ts-task-active` nags when
  current branch ≠ epic's `branch:` `` ended up rendered as `TASK_318["`/...`"]`,
  which mermaid's kanban grammar interprets as the start of a markdown-string
  label. The local pre-commit hook never ran `python scripts/validate_mermaid.py`,
  so the broken `KANBAN.md` regenerated by housekeep landed without anyone
  noticing.
- **Static Analysis / clang-tidy.** Pre-existing brace-style and
  redundant-decl violations in `lib/PedalLogic/src/*.cpp`. These had likely
  accumulated for some time — the pre-commit hook also did not invoke
  `clang-tidy`, so `WarningsAsErrors: "*"` only fired in CI.

**Wiring change that closed the gap:**

- `awesome-task-system/scripts/housekeep.py` now strips `` ` `` (in addition
  to `"`) from kanban labels. New regression test
  `test_kanban_label_strips_backticks` in
  `awesome-task-system/scripts/tests/test_housekeep.py`.
- `lib/PedalLogic/src/*.cpp` and `lib/PedalLogic/include/ble_config_reassembler.h`
  now pass clang-tidy clean with the project's existing `.clang-tidy` config:
  braces auto-applied via `clang-tidy --fix`; one-off fixes for
  `readability-redundant-declaration`, `performance-enum-size`,
  `clang-diagnostic-implicit-int-conversion`,
  `readability-function-cognitive-complexity`, and a NOLINT for two intentional
  `#define`s gated by `#ifndef`.
- `scripts/pre-commit` now runs `python scripts/validate_mermaid.py` whenever
  staged files include any `.md`, and `clang-tidy` (mirroring the CI command)
  whenever staged files include `lib/PedalLogic/src/*.cpp`. Missing tools are a
  loud failure with an install hint, not a silent skip — closing the
  silent-skip antipattern called out in the task description.
