## Description

Add an auto-generated burn-up section to `docs/developers/tasks/OVERVIEW.md`
showing cumulative tasks closed, cumulative epics closed, and cumulative
effort hours (estimated vs. actual) since the last git tag. The section is
delimited by `<!-- BURNUP:START -->` / `<!-- BURNUP:END -->` markers and
regenerated by `housekeep.py` on every `--apply` pass; idempotent (no-op when
the new block byte-matches the current one).

Lands last in EPIC-020. Both data sources are in place by then:

- TASK-269 — paused tasks live in `paused/`, so the closed-pathspec restriction
  (`closed/task-*.md`, `closed/epic-*.md`) correctly excludes them with no
  special-casing.
- TASK-270 — `effort_actual` is being written on close, so the actuals line in
  the chart is meaningful from day one.

New script: `awesome-task-system/scripts/release_burnup.py` (synced to
live per TASK-268).

Pseudocode:

```python
def main():
    tag = git("describe", "--tags", "--abbrev=0").strip()
    closed_files = parse_git_log_for_closures(tag)
    closures = []
    for path, date in closed_files:
        fm = parse_frontmatter(path)
        closures.append({
            "path": path,
            "date": date,
            "kind": "epic" if path.startswith("epic-") else "task",
            "effort_h": tshirt_to_hours(fm.get("effort")),
            "effort_actual_h": tshirt_to_hours(fm.get("effort_actual")),
        })
    closures = dedupe_by_basename_keep_latest(closures)
    series = aggregate_by_day(closures)
    block = render_burnup_block(series, tag)
    update_overview_section(block)  # byte-compare; only mutate if different
```

Canonical t-shirt → hours:

| Label | Hours |
|---|---|
| `XS (<30m)` | 0.25 |
| `Small (<2h)` | 1 |
| `Medium (2-8h)` | 5 |
| `Large (8-24h)` | 16 |
| `Extra Large (24-40h)` | 32 |
| `XXL (>40h)` | 56 (convention, not a true midpoint) |

Legacy labels (read-only, fade over time):

| Legacy label | Maps to |
|---|---|
| `Trivial (<30m)` | XS → 0.25 |
| `Small (1-2h)`, `Small (1-3h)`, `Small (2-4h)` | S → 1 |
| `Large (>8h)` | L → 16 |

Git-log parser:

```bash
git log <tag>..HEAD --diff-filter=A --no-renames --name-only \
  --date=short --pretty=format:'COMMIT %ad' \
  -- 'docs/developers/tasks/closed/task-*.md' \
     'docs/developers/tasks/closed/epic-*.md'
```

Stream parser: each `COMMIT <date>` sentinel sets the current commit date;
every following non-empty path until the next sentinel is a closure on that
date. Dedupe by basename, keeping the latest closure date per task (handles
`open → active → closed → reopened → closed` round-trips).

Rendered content:

1. Summary table (date / tasks closed / cum. tasks / est. h / cum. est. h /
   actual h / cum. actual h / epics closed / cum. epics).
2. Two side-by-side `xychart-beta` blocks: cumulative tasks, cumulative epics
   (two charts because `xychart-beta` has no secondary y-axis).
3. One `xychart-beta` block underneath: cumulative effort hours, two
   color-distinguished lines (estimate, actual). Markdown legend below the
   chart with the color codes. Encode series identity in the chart title too
   ("first line: estimate, second: actual") so the chart degrades gracefully
   when `themeVariables.xyChart.plotColorPalette` is not honored.

## Notes

- **Mermaid table-cell rendering** — used for the side-by-side cumulative-tasks / cumulative-epics charts. GitHub renders Mermaid in table cells today; if a future change breaks it (or if rendering moves to MkDocs Material per IDEA-022), the fallback is two stacked charts with their own headings. Keep the fallback documented in the script.
- **`themeVariables.xyChart.plotColorPalette` may not be honored everywhere.** Mitigation: encode series identity in the chart title (e.g. "Cumulative effort hours — first line: estimate, second: actual"). Ugly if colors render correctly, accurate if they don't.
- **Idempotency is non-negotiable.** Build the new block in memory, byte-compare against the current `BURNUP:START`/`BURNUP:END` slice, only mutate the file if they differ. Otherwise `housekeep --apply` becomes a noisy commit-spammer on idle days.
- **Dedupe-by-basename + `--no-renames`** is the contract that handles reopen/reclose round-trips and file moves between `open/` ↔ `active/` ↔ `paused/` ↔ `closed/`. Keep the unit tests honest about this — synthetic git-log fixtures are the right level.
- Effort **M (2–8h)**. Script itself is straightforward; most of the time goes into Mermaid rendering quirks and the housekeep integration.
