# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest stable | ✅ |
| Previous stable | ✅ |
| Older versions | ❌ |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report vulnerabilities privately via GitHub's built-in security advisory feature:

1. Go to the [Security tab](https://github.com/tgd1975/AwesomeStudioPedal/security)
2. Click **Report a vulnerability**
3. Fill in the details

### Response Timeline

| Stage | Target |
|-------|--------|
| Acknowledgement | Within 24 hours |
| Triage & severity assessment | Within 72 hours |
| Fix or mitigation | Depends on severity |
| Public disclosure | After fix is released |

## Disclosure Policy

- Vulnerabilities are kept confidential until a fix is available
- A public advisory is published alongside the fix release
- Credit is given to the reporter unless anonymity is requested
