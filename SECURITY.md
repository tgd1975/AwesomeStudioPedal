# Security Policy

## Project Context

AwesomeStudioPedal is an open-source hobby project maintained primarily by a single developer. The project follows a "best effort" approach to security, with the following considerations:

- This is a personal, non-commercial project with limited resources
- The only security-relevant component is the BLE keyboard functionality
- Response times and fixes depend on the maintainer's availability and discretion

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest stable | ✅ |
| Previous stable | ❌ |
| Older versions | ❌ |

## Reporting a Vulnerability

Security reports are welcome and can be submitted via GitHub's security advisory feature:

1. Go to the [Security tab](https://github.com/tgd1975/AwesomeStudioPedal/security)
2. Click **Report a vulnerability**
3. Fill in the details

### Important Notes on Response

- **No guaranteed response time**: As a hobby project, responses depend on the maintainer's availability
- **No guaranteed fixes**: The maintainer reserves the right to prioritize or decline fixes based on impact and feasibility
- **Community contributions welcome**: Security improvements via pull requests are encouraged
- **BLE keyboard limitations**: The BLE keyboard functionality has inherent security limitations typical of HID devices

## Disclosure Policy

- Vulnerability reports will be acknowledged when possible
- Fixes will be implemented based on severity, impact, and available resources
- Public disclosure may occur after a fix is available, but is not guaranteed
- Credit will be given to reporters when appropriate and requested
